# Pixel Art Refiner 参数使用指南

## 参数总览

| 参数名 | 类型 | 默认值 | 取值范围 | 作用阶段 |
|--------|------|--------|----------|----------|
| 采样方法 | 枚举 | center | center/median/majority | 颜色采样 |
| 最小像素尺寸 | 浮点数 | 4.0 | 1.0 - 20.0 | 网格检测 |
| 峰宽 | 整数 | 6 | 1 - 20 | 网格检测 |
| 细化强度 | 浮点数 | 0.25 | 0.0 - 1.0 | 网格对齐 |
| 修正正方形 | 布尔值 | true | true/false | 输出调整 |

---

## 详细说明

### 1. 采样方法 (sample_method)

**作用**：决定如何从每个像素单元中提取颜色值。

**可选值**：
- **center (中心采样)**：取像素单元中心点的颜色。速度快，适合像素边界清晰的图像。
- **median (中位数采样)**：取单元内所有像素的中位数颜色。抗噪性好，适合有轻微模糊的图像。
- **majority (多数采样)**：使用 K-means 聚类，选取占比最多的颜色。适合有渐变或抗锯齿的像素图。

**使用建议**：
- 像素边界清晰 → 选 `center`
- 有轻微模糊/噪点 → 选 `median`
- AI生成的伪像素图 → 选 `majority`

---

### 2. 最小像素尺寸 (min_size)

**作用**：设置算法能识别的最小像素块尺寸（像素）。

**工作原理**：
算法通过频谱分析检测图像的周期性结构。`min_size` 用于过滤掉过小的周期：

```
如果检测到的像素大小 < min_size → 忽略该检测结果
```

**调整场景**：

| 问题现象 | 原因 | 解决方法 |
|----------|------|----------|
| 处理失败，提示"无法确定网格" | 检测到的周期太小被过滤 | 减小 `min_size`（如改为 2.0） |
| 输出网格过大，细节丢失 | 检测到了过大的周期 | 增大 `min_size`（如改为 8.0） |

**典型值**：
- 像素游戏素材（32x32）：4.0 - 8.0
- 高清像素艺术（64x64+）：8.0 - 12.0
- 小尺寸像素图（16x16）：2.0 - 4.0

---

### 3. 峰宽 (peak_width)

**作用**：控制 FFT 频谱分析中峰值检测的灵敏度。

**工作原理**：
算法对图像进行快速傅里叶变换（FFT），在频谱中寻找代表像素周期的峰值。`peak_width` 定义了检测峰值时考虑的邻域宽度：

- **值较小（1-3）**：更严格的峰值检测，只识别明显的周期
- **值较大（10-20）**：更宽松的检测，可能检测到更多噪声

**调整场景**：

| 问题现象 | 解决方法 |
|----------|----------|
| 处理失败，无法检测周期 | 适当减小 `peak_width` |
| 检测结果不稳定，每次不同 | 增大 `peak_width` 平滑噪声 |

**典型值**：4 - 8

---

### 4. 细化强度 (refine_intensity)

**作用**：控制网格线对齐到图像边缘的强度。

**工作原理**：
算法首先根据检测到的周期生成初始网格，然后搜索图像中的实际边缘（通过 Sobel 梯度检测）来微调网格线位置。

```
搜索范围 = 像素尺寸 × refine_intensity
```

- **0.0**：不进行细化，使用理论计算的网格位置
- **0.25**（默认）：在 ±25% 像素尺寸范围内搜索最佳边缘
- **0.5+**：大范围搜索，网格线可能明显偏移

**调整场景**：

| 问题现象 | 解决方法 |
|----------|----------|
| 像素块边缘对齐不准确 | 增大 `refine_intensity` |
| 网格线偏移过多，变形严重 | 减小 `refine_intensity` |
| 像素块位置规律，但边缘模糊 | 适当增大 `refine_intensity` |

**典型值**：0.2 - 0.4

---

### 5. 修正正方形 (fix_square)

**作用**：当检测到接近正方形的图像时，强制输出完美正方形。

**工作原理**：
当 `|宽度 - 高度| == 1` 时：
- 如果开启：通过删除或复制行列使输出成为正方形
- 如果关闭：保持原样输出

**使用建议**：
- 处理游戏精灵图、图标等正方形素材 → 保持 `true`
- 处理矩形像素图（如横幅、背景）→ 设为 `false`

## 故障排除

### 问题："无法确定图像的像素网格大小"

**可能原因与解决方法**：

1. **像素尺寸过小**
   - 尝试减小 `min_size` 到 2.0 或 1.0

2. **图像不是像素风格**
   - 确保输入是像素艺术图像，而不是普通照片

3. **像素周期不明显**
   - 尝试调整 `peak_width`（4-10 之间尝试）

4. **图像分辨率与像素尺寸不匹配**
   - 确保图像没有被过度缩放

### 问题：输出图像变形/比例不对

**可能原因与解决方法**：

1. **X/Y 方向像素大小不一致**
   - 检查原始图像是否被非等比例缩放
   - 尝试调整 `min_size` 过滤异常周期

2. **接近正方形时强制修正**
   - 关闭 `fix_square` 参数

### 问题：像素边缘不对齐

**可能原因与解决方法**：

1. **细化强度不足**
   - 增大 `refine_intensity` 到 0.3-0.5

2. **采样方法不适合**
   - 尝试 `median` 或 `majority` 采样

## 参数交互关系

```
图像输入
    │
    ▼
┌─────────────────┐
│  网格检测阶段   │ ← min_size, peak_width 影响检测
│  (FFT/梯度分析) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  网格对齐阶段   │ ← refine_intensity 影响细化
│  (边缘搜索)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  颜色采样阶段   │ ← sample_method 影响颜色
│  (像素值提取)   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  输出调整阶段   │ ← fix_square 影响最终尺寸
│  (尺寸修正)     │
└─────────────────┘
```

## 高级技巧

### 技巧 1：手动指定网格大小

如果自动检测失败，可以通过 API 直接指定网格大小：

```python
from perfect_pixel import get_perfect_pixel

# 手动指定 32x32 的网格
w, h, output = get_perfect_pixel(
    image,
    grid_size=(32, 32),  # 强制使用 32x32 网格
    sample_method="center"
)
```

### 技巧 2：批量处理时的参数优化

对于同一批风格相似的图像，先测试几张找到最佳参数，然后批量应用。

### 技巧 3：调试模式

在代码中开启 `debug=True` 查看网格检测过程：

```python
w, h, output = get_perfect_pixel(image, debug=True)
```

这会显示网格线叠加的图像，帮助你理解参数效果。
